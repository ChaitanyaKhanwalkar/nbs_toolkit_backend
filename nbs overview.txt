Project Overview App Name: NbS Toolkit Goal: To recommend nature-based water treatment solutions (NbS), plants, and detailed implementation plans for Indian districts, based on water type, user-uploaded data, and region-specific data. Final Confirmed App/User Flow User opens app → Splash screen Home Screen: Choose between: Uploading water quality data (CSV/Excel) Or selecting water type manually If uploading: User picks a file and state, backend classifies water and contaminants If selecting water type: User selects water type and state Results: App recommends ranked plants and NbS solutions for that region, based on water type, contaminants, and district/state User taps an NbS/plant for full details and implementation plan 2. Backend Code (Latest, by Module) Module 1: Water Type Classification from Data Endpoint: /water-data (POST) Input: File upload (CSV/XLSX) and state name Output: Classified water type, contaminants, recommended plants/NbS Logic: (In water_data.py) Reads uploaded file, parses for key contaminants (using defined thresholds) Classifies water type (grey, black, yellow, brown) by content Returns to frontend # app/api/water_data.py from fastapi import APIRouter, UploadFile, Form, Depends, HTTPException from sqlalchemy.orm import Session import pandas as pd from app.db.database import SessionLocal from app.utils.recommendation_utils import get_recommendation_data router = APIRouter() def get_db(): db = SessionLocal() try: yield db finally: db.close() @router.post("/water-data") async def upload_water_data(file: UploadFile, state_name: str = Form(...), db: Session = Depends(get_db)): df = pd.read_csv(file.file) # Simple logic: classify by columns/thresholds (customize as needed) if "COD" in df.columns and df["COD"].mean() > 250: water_type = "Black Water" elif "TSS" in df.columns and df["TSS"].mean() > 100: water_type = "Grey Water" else: water_type = "Brown Water" # Continue as with manual input data = get_recommendation_data(state_name, water_type, db) data["classified_water_type"] = water_type return data Module 2: Contaminant Profiling (Integrated into the water data upload above. Can also add a separate /contaminants if needed.) Logic: Extracts relevant contaminant levels (e.g. BOD, COD, TSS, Nitrate) and classifies or flags high values Output: List of contaminants, optional health warning Module 3: Plant Filtering Logic: In recommendation_utils.py, plants are filtered by: State name Water type Soil type (from merged district data) Climate preference Optionally: contaminant tolerance (if implemented) # See recommendation_utils.py below for get_matches() function Module 4: NbS Filtering Logic: In recommendation_utils.py, NbS filtered by: Water type State/district (if applicable) Soil/climate suitability Module 5: NbS Ranking Logic Algorithm: Perfect match: state + water + soil State + water Water only Fuzzy soil match (with fuzzywuzzy) Any/All (fallback) Returns first 5, with match_level field Custom: Plants/NbS are ranked and sorted by best match, then by order in CSV Module 6: Implementation Plan API Endpoint: /implementation (GET) Input: solution (NbS solution name) Output: implementation_steps, maintenance_requirements (as string or array) Logic: Case-insensitive match on solution name # app/api/implementation.py from fastapi import APIRouter, Query, Depends, HTTPException from sqlalchemy.orm import Session from app.db.database import SessionLocal import pandas as pd router = APIRouter() def get_db(): db = SessionLocal() try: yield db finally: db.close() @router.get("/implementation") def get_implementation(solution: str = Query(...), db: Session = Depends(get_db)): df = pd.read_sql("SELECT * FROM nbs_implementation", db.bind) record = df[df['solution'].str.lower() == solution.lower()] if record.empty: raise HTTPException(status_code=404, detail="No implementation plan found for solution.") record = record.iloc[0] return { "solution": record['solution'], "implementation_steps": record['implementation_steps'], "maintenance_requirements": record['maintenance_requirements'] } Module 7: Fallback Logic (No Data Upload) If user skips file upload, can select water type and state directly. Endpoint: /recommendations (GET) Input: state_name, water_type Output: List of plants, NbS options, with best matches first # app/api/recommendations.py from fastapi import APIRouter, Query, Depends from sqlalchemy.orm import Session from app.db.database import SessionLocal from app.utils.recommendation_utils import get_recommendation_data from fastapi.responses import JSONResponse router = APIRouter() def get_db(): db = SessionLocal() try: yield db finally: db.close() @router.get("/recommendations") def get_recommendations(state_name: str, water_type: str, db: Session = Depends(get_db)): data = get_recommendation_data(state_name, water_type, db) if not data or not isinstance(data, dict): return JSONResponse(content={"plants": [], "nbs_options": []}) data.setdefault("plants", []) data.setdefault("nbs_options", []) return JSONResponse(content=data) Backend Utility: recommendation_utils.py # app/utils/recommendation_utils.py import pandas as pd from fuzzywuzzy import fuzz def get_recommendation_data(state_name, water_type, db): merged_df = pd.read_sql("SELECT * FROM merged_district_data", db.bind) merged = merged_df[merged_df['state_name'].str.lower() == state_name.lower()] soil_type = merged.iloc[0]['soil_type'] if not merged.empty else None plant_df = pd.read_sql("SELECT * FROM plant_data", db.bind) nbs_df = pd.read_sql("SELECT * FROM nbs_options", db.bind) def get_matches(df, key): results = pd.DataFrame() match_levels = [] # Perfect if soil_type: perfect = df[(df['state_name'].str.lower() == state_name.lower()) & (df['optimal_water_type'].str.lower() == water_type.lower()) & (df['soil_type'].str.lower() == str(soil_type).lower())] results = pd.concat([results, perfect]) match_levels.extend(['Perfect'] * len(perfect)) # State+Water if len(results) < 5: state_water = df[(df['state_name'].str.lower() == state_name.lower()) & (df['optimal_water_type'].str.lower() == water_type.lower())] new_state_water = state_water[~state_water.index.isin(results.index)] results = pd.concat([results, new_state_water]) match_levels.extend(['State+Water'] * len(new_state_water)) # Water Only if len(results) < 5: water = df[df['optimal_water_type'].str.lower() == water_type.lower()] new_water = water[~water.index.isin(results.index)] results = pd.concat([results, new_water]) match_levels.extend(['Water Only'] * len(new_water)) # Fuzzy (soil) if len(results) < 5: if soil_type: df['match_score'] = df.apply(lambda row: fuzz.token_set_ratio( str(row.get('soil_type', '')).lower(), str(soil_type).lower()), axis=1) fuzzy = df[~df.index.isin(results.index)].sort_values('match_score', ascending=False) results = pd.concat([results, fuzzy]) match_levels.extend(['Fuzzy'] * len(fuzzy)) else: others = df[~df.index.isin(results.index)] results = pd.concat([results, others]) match_levels.extend(['Any'] * len(others)) results = results.head(5) match_levels = match_levels[:5] data = results.to_dict(orient='records') for i, item in enumerate(data): item['match_level'] = match_levels[i] if i < len(match_levels) else 'Any' return data, match_levels[0] if match_levels else 'None' plants, plant_level = get_matches(plant_df, 'plants') nbs, nbs_level = get_matches(nbs_df, 'nbs_options') return { "plant_match_level": plant_level, "nbs_match_level": nbs_level, "soil_type": soil_type, "plants": plants, "nbs_options": nbs } 5. Database Design Tables: merged_district_data plant_data water_data nbs_options nbs_implementation Schema (SQL, for PostgreSQL): -- merged_district_data CREATE TABLE merged_district_data ( id SERIAL PRIMARY KEY, state_name TEXT, district_name TEXT, soil_type TEXT ); -- plant_data CREATE TABLE plant_data ( id SERIAL PRIMARY KEY, plant_species TEXT, climate_preference TEXT, water_needs TEXT, ecological_role TEXT, soil_type TEXT, locational_availability TEXT, pollution_tolerance TEXT, state_name TEXT, optimal_water_type TEXT ); -- nbs_options CREATE TABLE nbs_options ( id SERIAL PRIMARY KEY, solution TEXT, optimal_water_type TEXT, location_suitability TEXT, climate_suitability TEXT, soil_type TEXT, resource_requirements TEXT, notes TEXT, state_name TEXT ); -- nbs_implementation CREATE TABLE nbs_implementation ( id SERIAL PRIMARY KEY, solution TEXT, implementation_steps TEXT, maintenance_requirements TEXT ); Relationships: solution in nbs_implementation matches (exact string) solution in nbs_options. CSV Import: Used pgAdmin "Import/Export Data" tool, mapped CSV columns to the above schemas. 6. CSV & Data Mapping merged_district_data.csv → merged_district_data (state, district, soil_type) plant_data.csv → plant_data (species, preferences, tolerance, etc.) nbs_options.csv → nbs_options (solution, suitability, resource req, etc.) nbs_implementation.csv → nbs_implementation (solution, steps, maintenance) water_data.csv → water_data (all water chemistry columns for upload/classification) Transformations: Ensure all solution fields match (no typos/extra spaces/case mismatches). id fields are auto-incremented. All columns as TEXT except id. 7. Logic Flow (Step by Step) A. User Uploads Water Data User uploads CSV, selects state. Backend parses file, classifies water type by logic (thresholds, mean/median). Calls main recommendation function with state and classified water type. Returns ranked plant and NbS options for display. B. User Selects Water Type User selects water type and state manually. Calls recommendation endpoint. Returns ranked plant and NbS options for display. C. Contaminant Identification During upload, backend checks specific columns (COD, TSS, etc.) for threshold breaches and flags water type/contaminant profile. D. Filtering & Ranking Matches are filtered by state, water type, soil type (if available). Fallback logic ensures 5 results using broader matches (see above). 8.
